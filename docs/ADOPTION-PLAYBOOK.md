# Adoption Playbook (Pre-Build Stage)

## Objective
Maximize credible early adoption signals before production infrastructure exists.

## Positioning
- ACP is an **open protocol specification**, not a payment processor.
- Current milestone is **validation**, not production deployment.
- The immediate ask is schema and interoperability feedback.

## High-Leverage Moves
1. Publish clear scope boundaries so builders trust the roadmap.
2. Prioritize implementer friction over marketing polish.
3. Collect structured feedback with labels and decision logs.
4. Ship conformance criteria before advanced feature expansion.

## Partner Profile (Now)
- Teams already building multi-agent workflows
- Infra teams who need escrow-like settlement semantics
- Researchers focused on agent governance, verification, and replay

## Messaging Guardrails
- Avoid claiming production readiness before it exists.
- Separate protocol guarantees from implementation aspirations.
- Be explicit about what is draft, in-progress, and locked.

## Near-Term Outputs
- Public validation reports
- Edge-case catalog
- Draft compatibility policy
- Conformance scope document
